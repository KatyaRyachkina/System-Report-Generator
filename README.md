# Основная суть проекта:

Генератор системных отчетов - инструмент для автоматизированного сбора и анализа информации о состоянии компьютера, который предоставляет подробные данные о системе, аппаратном обеспечении и программном окружении в удобном формате.

# Что делает программа?

Собирает информацию со всех компонентов системы

Анализирует состояние - проверяет процессор, память, диски, сеть и процессы

Представляет данные - форматирует информацию для удобного восприятия

Сохраняет результаты - записывает отчеты в файлы для дальнейшего анализа

Универсально работает - поддерживает различные операционные системы

# Что МОЖЕТ этот генератор отчетов:

Собирать информацию о платформе и ОС

Анализировать загрузку процессора и памяти

Проверять дисковое пространство и ввод-вывод

Отслеживать сетевые интерфейсы и статистику

Показывать активные процессы и пользователей

Экспортировать данные в текстовом и JSON формате

Автоматически сохранять отчеты с временными метками

# Вывод

Для мониторинга системы - мощный и удобный инструмент, который предоставляет полную картину состояния компьютера, помогает в диагностике проблем и оптимизации производительности, используя современные библиотеки для сбора системной информации и гибкие форматы вывода данных.

# Пример использования генератора 1.

# Создание текстового отчета с выводом в консоль и сохранением в файл

bash

python system_report.py --format text --print --output system_status

# Пример использования генератора 2.

# Создание JSON отчета для автоматической обработки другими программами

bash

python system_report.py --format json --output system_data

# Пример использования генератора 3.

# Быстрая проверка системы с выводом только в консоль

bash

python system_report.py --print

#!/bin/bash
# Ежедневный сбор системной информации
python system_report.py --format json -o /var/log/system_reports/$(hostname)_$(date +%Y%m%d).json

КРАТКО:

- `--format`, выбор формата: `text` (читаемый) или `json` (машиночитаемый)
    
- `-o`, `--output`, имя выходного файла (без расширения)
    
- `-p`, `--print`, вывод отчета в консоль
    

# Что собирает программа:

## Платформа и ОС

- Название операционной системы и версия
    
- Имя хоста и основной IP-адрес
    
- Время создания отчета
    

## Процессор (CPU)

- Количество физических и логических ядер
    
- Текущая частота процессора
    
- Общая загрузка и загрузка по каждому ядру
    
- Температура (если доступны датчики)
    

## Память (RAM и Swap)

- Общий объем оперативной памяти
    
- Используемая память в GB и процентах
    
- Информация о файле подкачки
    

## Дисковые накопители

- Информация о всех разделах и точках монтирования
    
- Типы файловых систем
    
- Общий, использованный и свободный объем
    
- Статистика ввода-вывода (чтение/запись)
    

## Сеть

- Общая сетевая статистика (отправлено/получено)
    
- Информация о всех сетевых интерфейсах
    
- IP-адреса (IPv4 и IPv6) и MAC-адреса
    
- Статус интерфейсов и скорость соединения
    

## Процессы и пользователи

- Топ-8 процессов по использованию памяти
    
- Имя процесса, PID, загрузка CPU и памяти
    
- Активные пользователи системы
    
- Время входа пользователей
    

## Системная информация

- Время последней загрузки системы
    
- Данные с датчиков температуры
    
- Время работы системы
    

# Форматы вывода:

## Текстовый формат (--format text)

Удобен для быстрого просмотра человеком. Содержит:

- Структурированные разделы с заголовками
    
- Форматированные таблицы
    
- Проценты и единицы измерения
    
- Четкое визуальное разделение
    

## JSON формат (--format json)

Идеален для автоматической обработки. Содержит:

- Структурированные данные в машиночитаемом формате
    
- Все числовые значения в исходном виде
    
- Вложенные структуры данных
    
- Возможность парсинга другими программами
